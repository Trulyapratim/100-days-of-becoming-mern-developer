Let's Try to understand the Execution Context in Js.

We are learnig execution Context because in JavaScript everything happens inside an execution context.JavaScript is a synchronous single threaded language.

Execution context is an abstract concept that holds information about the environment within which the current code is being executed. It contains variables, functions, and the value of this keyword. There are three types of execution contexts: global, function, and eval.

let's understand the whole concept through some codes.

var ans = num * num;
return ans;
}

I hope you all have been able to follow up till now. Let's try to understand the execution context.

So, as we have already talked about that everything in javaScript happens inside an execution Context.Whenever we write an javaScript code an Global execution context get's created.

Global execution context is the default or base execution context where your code starts to execute. It creates a global object (window in browsers) and a special object called 'this'. The global execution context is created when the JavaScript engine starts executing your code and it remains until the program finishes.

Memory/Variable environment Code/Thread of Execution
This section gets created when we write our js code. This is the phase in which js code start executing.
In creation phase : Memory space of all variables and functions gets allocated. For variable: Undefined and for function(){ ...} entire function codes get allocated. Now in this phase it will start replacing undefined and start putting real values to the variable. And when function gets invoked a new execution context get created in thread of execution and then again this process starts. And then this new execution context gets deallocated from the memory.

Arrow function behaves like a variable in Gloabal Execution context. Means in creation phase it gets assigned with undefined instead of whole function code.

During this whole process a call stack also gets created. Call Stack maintains the order of execution of execution context.
